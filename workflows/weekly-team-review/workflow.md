# Weekly Team Review Workflow

## Purpose

Create a manager-ready weekly operating review from daily report evidence, optional client notes, and optional manager notes.

Preferred skill:
`skills/weekly-team-review/`

## Default Outputs

Save outputs under:
`outputs/weekly/YYYY-MM-DD_YYYY-MM-DD/`

Required files:

- `weekly-team-review.md`
- `weekly-team-review.html`

The HTML file is the default manager-facing artifact. It must be standalone and open directly in a browser. Do not create PDF or PowerPoint outputs unless Mike explicitly requests that format for the run.

## Required Review Flow

1. Resolve weekly date window.
2. Collect available daily-report evidence for the covered window.
3. Add optional client notes or manager notes when provided.
4. Group findings by client/team first, then employee.
5. Identify repeated blockers, missing-report patterns, low-visibility signals, delivery risk, capacity pressure, and strong performer signals.
6. Separate direct evidence, interpretation, assumptions, and missing data.
7. Create the Markdown audit/source draft.
8. Create the standalone HTML dashboard.
9. Verify the HTML includes search, filters, navigation, client/team sections, employee summaries, manager actions, and source audit.

## Review Gate

Default status: `Draft for Mike review`.

Any client-facing, employee-facing, compliance, or performance-management follow-up language must remain a draft until Mike approves it.
